description: "Full reference for /plot — claim and sell plots, set plot types, manage permissions, create plot groups and districts, and configure individual plot toggles."
---

The `/plot` command controls everything related to individual TownBlocks (plots). Residents use it to claim and personalise plots they own, while mayors use it to manage unowned town land. Plot types, permission flags, groups, and districts are all configured here.

<Note>
You must be standing inside the plot you want to modify. Most subcommands require you to be either the plot owner or the town mayor. Permission nodes follow the pattern `towny.command.plot.<subcommand>`.
</Note>

## Claiming and releasing plots

```bash
# Claim the plot under your feet
/plot claim

# Release your ownership of the plot
/plot unclaim
```

<Tip>
Use `/plot claim` when standing on a for-sale plot to purchase it at the listed price. If the plot is unowned, it costs the town's default plot price.
</Tip>

## Selling plots

```bash
# List the current plot for 100
/plot forsale 100

# Take it off the market
/plot notforsale

# Mayor: evict the current resident from their plot
/plot evict
```

## Plot permissions

Towny uses a permission matrix: rows are resident types (`resident`, `nation`, `ally`, `outsider`) and columns are action types (`build`, `destroy`, `switch`, `item_use`).

```bash
# Open the permissions HUD overlay
/plot perm hud

# Open the GUI editor
/plot perm gui

# Grant build access to a friend
/plot perm add build Steve

# Remove switch access from outsiders
/plot perm remove switch outsider

# Set permissions via shorthand flags
/plot set perm resident build on
```

## Plot types

Setting a plot type changes the rules and taxes that apply to it. Only the mayor or plot owner with appropriate permission can change types.

<AccordionGroup>
  <Accordion title="Available plot types">

| Type | Description |
|---|---|
| Default | Standard town plot with normal permissions. |
| Arena | PvP is always enabled; death penalties may be waived. |
| Bank | Used by economy add-ons for banking. |
| Embassy | Can be purchased by non-residents for a separate price. |
| Farm | Restricts building to farm-related blocks. |
| Inn | Allows any player to use a bed and set a spawn. |
| Jail | Cells used to hold jailed residents. |
| Shop | Taxed at the shop rate; used for player shops. |
| Spleef | Designated spleef arena plot. |
| Wilds | Plot behaves like the wilderness (unprotected). |

  </Accordion>
</AccordionGroup>

```bash
# Convert to an embassy (purchasable by outsiders)
/plot set embassy

# Convert to a farm
/plot set farm

# Convert to a shop
/plot set shop

# Reset back to a normal town plot
/plot set reset
```

## Plot toggles

Toggles override town-level defaults for a single plot.

```bash
# Allow explosions on this plot only
/plot toggle explosion on

# Disable PvP on this plot
/plot toggle pvp off

# Prevent fire from spreading
/plot toggle fire off

# Disable hostile mob spawning
/plot toggle mobs off
```

---

## Plot groups

Plot groups let you manage multiple plots as a single unit.

```bash
# Add the current plot to a group named "market"
/plot group add market

# Set the type for all "market" plots at once
/plot group set shop

# List all for-sale at price 200
/plot group forsale 200

# Remove all from sale
/plot group notforsale

# Remove this plot from its group
/plot group remove

# Delete the entire group
/plot group delete market
```

---

## Districts

Districts are named zones within a town — useful for organising neighbourhoods or themed areas.

```bash
# Add the current plot to the "harbour" district
/plot district add harbour

# Remove this plot from its district
/plot district remove

# Rename the district
/plot district rename harbour docks

# Delete the district
/plot district delete docks
```

---

## Trusted players

Trusted players get full build/destroy/switch/item-use access to the plot regardless of standard permission flags.

```bash
# Trust Steve on the current plot
/plot trust add Steve

# Remove Steve's trusted access
/plot trust remove Steve
```
