The Towny Developer API lets you integrate directly with Towny's data model and event system from your own Bukkit or Paper plugin. You can look up residents, towns, and nations by name or UUID; react to in-game changes via Bukkit events; attach persistent custom data to any Towny object; and inject new subcommands into `/town`, `/nation`, and other built-in commands.

All API classes live in the `com.palmergames.bukkit.towny` package and are distributed through JitPack, so no private repository access is required. The primary entry point is the `TownyAPI` singleton; the internal data store is accessible through `TownyUniverse`.

## What the API offers

- **Object lookup** — fetch `Resident`, `Town`, `Nation`, `TownBlock`, and `TownyWorld` objects by name, UUID, location, or player reference.
- **Event system** — listen to dozens of cancellable and non-cancellable Bukkit events covering every major Towny action.
- **Custom metadata** — attach typed key-value data to any Towny object and have it automatically persisted.
- **Command addons** — inject your own subcommands into Towny's command tree without touching Towny's source code.
- **Tab completion helpers** — static utilities for building tab completers that return resident, town, and nation names.

## Who this is for

Any developer building a Bukkit or Paper plugin that needs to interact with Towny's towns, nations, or land-claim system. You should be comfortable writing a basic Bukkit plugin before using this API.

<Note>
Towny requires **Java 17** and targets the Paper API. Your plugin must list Towny as a dependency in `plugin.yml` to guarantee Towny is loaded first.
</Note>

Ready to add Towny to your project? Start with the [setup guide](/api/setup).
