title: "Towny events: listening for in-game changes"
description: "Categorised reference of Towny's Bukkit events, whether each is cancellable, and a code example showing how to listen with the @EventHandler annotation."
---

Towny fires standard Bukkit events for every major action — creating a town, claiming land, adding residents, performing bank transactions, and more. You listen to them exactly as you would any other Bukkit event: implement `Listener`, annotate a method with `@EventHandler`, and register your listener in `onEnable`.

```java
import com.palmergames.bukkit.towny.event.NewTownEvent;
import org.bukkit.event.EventHandler;
import org.bukkit.event.Listener;

public class TownyListener implements Listener {

    @EventHandler
    public void onNewTown(NewTownEvent event) {
        Town town = event.getTown();
        Bukkit.broadcastMessage("A new town was founded: " + town.getName());
    }
}
```

Register in your `onEnable`:

```java
getServer().getPluginManager().registerEvents(new TownyListener(), this);
```

<Note>
Many Towny events run **asynchronously** (the class constructor passes `!Bukkit.getServer().isPrimaryThread()` to the super call). Never call synchronous Bukkit API methods — including `entity.teleport()` — from an async event handler without scheduling back to the main thread first.
</Note>

## Listening to `NewTownEvent` — full example

```java
import com.palmergames.bukkit.towny.event.NewTownEvent;
import com.palmergames.bukkit.towny.object.Town;
import org.bukkit.Bukkit;
import org.bukkit.event.EventHandler;
import org.bukkit.event.EventPriority;
import org.bukkit.event.Listener;
import org.bukkit.plugin.java.JavaPlugin;

public class TownFoundListener implements Listener {

    private final JavaPlugin plugin;

    public TownFoundListener(JavaPlugin plugin) {
        this.plugin = plugin;
    }

    // EventPriority.MONITOR means we are only observing, not modifying.
    @EventHandler(priority = EventPriority.MONITOR)
    public void onNewTown(NewTownEvent event) {
        Town town = event.getTown();

        // NewTownEvent is async — schedule any synchronous work on the main thread.
        Bukkit.getScheduler().runTask(plugin, () -> {
            plugin.getLogger().info("New town created: " + town.getName()
                + " (mayor: " + town.getMayor().getName() + ")");
        });
    }
}
```
