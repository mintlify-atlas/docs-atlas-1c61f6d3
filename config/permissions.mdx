---
title: "Towny permission nodes reference"
description: "A complete reference of Towny's permission nodes — how they work with LuckPerms or Vault, the top-level nodes, and per-command nodes grouped by command."
---

Towny integrates with any permission plugin that supports the Bukkit permissions API. The most common choices are **LuckPerms**, **Vault** (bridging to other plugins), and **GroupManager**. Towny checks the standard Bukkit permission system, so any compatible plugin will work.

Towny ships with two separate permission layers:

1. **plugin.yml nodes** — standard Bukkit permission nodes that your permission plugin manages. These control command access and override abilities.
2. **townyperms.yml nodes** — granted to players based on their Towny rank (nomad, resident, mayor, king, and custom ranks). These are applied by Towny itself, not by your permission plugin.

This page covers the plugin.yml nodes. For `townyperms.yml`, see [townyperms.yml: configuring rank permissions](/config/townyperms).

---

## Top-level nodes

| Node | Default | Description |
|---|---|---|
| `towny.admin` | op | Full admin access. Grants all command nodes, build anywhere, and all spawn nodes. |
| `towny.town.resident` | false | Required to join a town. Included in `nomad` section of `townyperms.yml` by default. |
| `towny.townless` | false | Assigned to players who do not belong to a town. Use this to grant or restrict townless players. |
| `towny.wild.*` | false | Build, destroy, switch, and use items in the wilderness. |
| `towny.claimed.*` | false | Build, destroy, switch, and use items in all claimed town land. |
| `towny.bypass_death_costs` | false | The player is not charged Towny death costs. |
| `towny.bypass_bed_restriction` | false | The player is not subject to bed-use restrictions. |
| `towny.receives-plot-notifications` | true | Show border-crossing notifications when entering or leaving towns and plots. |

### Wilderness override nodes

`towny.wild.*` grants all four action types in the wilderness. You can grant or deny individual sub-nodes for finer control:

| Node | Description |
|---|---|
| `towny.wild.build.*` | Build in the wilderness |
| `towny.wild.destroy.*` | Break blocks in the wilderness |
| `towny.wild.switch.*` | Use switches (levers, buttons, doors) in the wilderness |
| `towny.wild.item_use.*` | Use items in the wilderness |

Replace `*` with a block ID (e.g. `towny.wild.build.stone`) to target specific materials.

### Claimed land override nodes

`towny.claimed.*` grants access across **all** claimed town land. Sub-nodes let you target specific zones:

| Node | Zone |
|---|---|
| `towny.claimed.townowned.*` | Town-owned (unassigned) plots |
| `towny.claimed.owntown.*` | Plots inside the player's own town |
| `towny.claimed.alltown.*` | Any claimed plot in any town |

Each zone supports `.build.*`, `.destroy.*`, `.switch.*`, and `.item_use.*` suffixes.

## Spawn nodes

| Node | Description |
|---|---|
| `towny.town.spawn.*` | All spawn travel nodes |
| `towny.town.spawn.town` | Use `/town spawn` to your own town |
| `towny.town.spawn.nation` | Use `/town spawn [town]` to another town in your nation |
| `towny.town.spawn.ally` | Use `/town spawn [town]` to an allied nation's town |
| `towny.town.spawn.public` | Use `/town spawn [town]` to any public town |
| `towny.town.spawn.outpost` | Use `/town outpost` |
| `towny.admin.spawn` | Admin spawn without warmup, cooldown, or charge |

## Admin-only nodes

| Node | Description |
|---|---|
| `towny.admin.nation_zone` | Build inside nation zones without restrictions |
| `towny.admin.outlaw.teleport_bypass` | Not teleported out of towns where the player is an outlaw |
| `towny.admin.town_commands.blacklist_bypass` | Bypass the town command blacklist |
| `towny.admin.updatealerts` | Receive update notifications on login |
| `towny.outlaw.jailer` | Required to jail outlaws killed inside a town |

---

<Tip>
In LuckPerms you can use **contexts** to apply Towny permissions only in certain worlds. For example, grant `towny.wild.*` only in a specific creative or freebuild world by creating a context-restricted permission entry.
</Tip>
